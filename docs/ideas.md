# Agent 层想法池

本文档用于先收集 AI 狼人杀 Agent 层相关想法。当前阶段不急于判断哪些一定能实现，也不急于做取舍；后续再根据实现难度、评分贡献、展示效果和实际验证结果筛选。

> **文档状态说明**：各章节已标注实现状态（截至 2026-06-03）。状态标签：✅ 已实现、⚠️ 部分实现、❌ 未实现、N/A 不适用。

## 1. 总体方向 ✅

本项目可以把狼人杀视为一个信息不对称、多角色协作与博弈的 Agent Team 测试环境。

Agent 层目标不是简单让模型返回一个动作，而是让每个角色 Agent 在有限可见信息下，基于记忆、局势判断、角色策略和博弈目标，做出可解释、可追踪、可复盘的决策。

核心关键词：

- 信息隔离
- 多角色策略差异
- 公共记忆与私有记忆
- 局势建模
- 自博弈数据收集
- 强推理决策
- 赛后复盘
- 长期策略沉淀

## 2. 自博弈与数据收集 ✅

可以先让 AI Agent 自己进行多局狼人杀博弈，把最终日志完整收集起来。

可收集内容：

- 每局完整对局日志
- 每个玩家身份
- 每轮发言
- 每次投票
- 每次技能使用
- 每个 Agent 的输入信息
- 每个 Agent 的输出动作
- 每次决策理由
- fallback 是否发生
- 最终胜负
- 关键转折点

这些数据后续可以用于：

- 分析 Agent 决策质量
- 提取成功/失败经验
- 调优 Prompt
- 对比不同 Agent 版本
- 生成复盘报告
- 构建 Leaderboard
- 做自进化或策略迭代

## 3. 强推理决策：ToT / GoT ❌

> **状态**：未实现。决策 pipeline 已简化为 6 步线性流程，无 ToT/GoT 推理内核。相关 step（`reason_with_tree_step`、`reason_with_graph_step`）和目录（`agent/reasoning/`）已从代码库中移除。

在关键决策上可以不优先考虑 token 成本，而是使用更强的推理方法。

### 3.1 Tree of Thoughts

对复杂局面生成多个候选策略，并分别推演后果。

示例流程：

1. 生成 3 到 5 个候选动作。
2. 分析每个动作的收益、风险和对阵营的影响。
3. 预测其他玩家可能反应。
4. 选择期望收益最高的动作。
5. 输出公开发言和私有理由。

适合使用 ToT 的场景：

- 预言家是否跳身份
- 女巫是否救人
- 女巫是否毒人
- 猎人是否开枪
- 狼人是否悍跳
- 狼人是否倒钩队友
- 白天归票
- PK 投票

### 3.2 Graph of Thoughts

可以把玩家关系、发言关系、投票关系、身份嫌疑建成图。

图中的节点：

- 玩家节点
- 角色节点
- 阵营节点
- 发言事件节点
- 投票事件节点
- 技能事件节点

图中的边：

- A 攻击 B
- A 保护 B
- A 投票 B
- A 跟随 B 的观点
- A 与 B 票型一致
- A 与 B 发言矛盾
- A 可能是 B 的队友

GoT 可以用于：

- 找狼坑
- 判断站边关系
- 判断狼人冲票
- 判断倒钩行为
- 推断神职身份
- 分析阵营协作

## 4. 分层记忆设计 ✅

Agent 记忆可以分成短期记忆、中期记忆、长期记忆和完整记忆。

### 4.1 短期记忆：当前这一局

短期记忆用于维持当前一局游戏内的动态上下文，支撑每次发言、投票、技能使用等即时决策。

可以维护一份结构化现场笔记：

```json
{
  "game_state": {
    "day": 2,
    "phase": "day_speech",
    "alive_players": [1, 2, 3, 5, 6, 8, 9, 10],
    "dead_players": [4, 7],
    "known_dead_roles": {
      "7": "hunter"
    }
  },
  "player_models": {
    "2": {
      "suspected_role": "werewolf",
      "wolf_probability": 0.72,
      "style_tags": ["aggressive", "logic_gap"],
      "behavior_records": ["follow_vote", "attacked_seer"]
    }
  },
  "key_events": [
    {
      "time": "day1_vote",
      "event": "2, 3, 4 voted 9",
      "impact": "possible wolf team coordination"
    }
  ],
  "self_strategy": {
    "role": "werewolf",
    "known_info": ["1 and 3 are wolf teammates"],
    "current_disguise": "villager",
    "speech_goal": "push suspicion onto 9"
  }
}
```

短期记忆可以包括：

- 当前天数和阶段
- 存活玩家列表
- 死亡玩家列表
- 已知死亡身份
- 每位玩家的身份猜测
- 每位玩家的狼人嫌疑度
- 每位玩家的发言风格
- 每位玩家的投票行为
- 关键事件流
- 自己当前身份
- 自己已知信息
- 当前伪装身份
- 当前回合发言目标

可以考虑使用后台 forked agent 维护结构化笔记。主 Agent 决策时只读取最新笔记，不需要自己从全量日志里重新整理。

### 4.2 中期记忆：每局结束后的经验卡片

中期记忆以“一局游戏结束”为触发点。

原因是单轮发言或单次行动很难判断好坏，只有整局结束后才能看到最终胜负和长期影响。

每局结束后可以提取经验卡片，按角色保存。

示例：

```json
{
  "game_id": "session_042",
  "role": "werewolf",
  "outcome": "lose",
  "summary": "首轮悍跳预言家失败，发言逻辑漏洞导致第一天被放逐。",
  "key_decisions": [
    {
      "context": "首轮警上，我是狼人，决定悍跳预言家",
      "action": "起跳预言家，给真预言家发查杀",
      "expected_outcome": "扰乱好人视野，扛推真预言家",
      "actual_result": "发言位置靠后，无法回应前置位质疑，逻辑矛盾被放大",
      "lesson": "狼人悍跳要考虑发言位置和队友铺垫，位置不好时优先隐狼。"
    }
  ],
  "reflections": [
    "低估了发言位置的重要性",
    "后续应谨慎选择起跳时机"
  ]
}
```

中期记忆可以沉淀：

- 本局身份
- 本局胜负
- 本局关键决策
- 预期收益
- 实际结果
- 成功原因
- 失败原因
- 下一局可复用经验

中期记忆建议按角色划分：

- 狼人经验池
- 预言家经验池
- 女巫经验池
- 猎人经验池
- 村民经验池
- 守卫经验池
- 白狼王经验池

### 4.3 长期记忆：跨多局策略沉淀

当中期记忆积累到 5 局以上，可以触发 dream / consolidate 流程。

长期记忆目标：

1. 合并相似经验。
2. 淘汰过时或低效策略。
3. 生成稳定策略原则。
4. 为新一局提供先验。

示例：

```text
狼人悍跳预言家需要满足：
1. 发言位置不能太靠后；
2. 队友最好提前铺垫；
3. 查杀目标要有可解释逻辑；
4. 被真预言家反打时要准备反驳链；
5. 如果局面没有队友支持，优先选择隐狼或倒钩。
```

长期记忆可以包括：

- 角色通用策略
- 常见失败模式
- 高收益打法
- 低收益打法
- 不同局势下的优先级
- 历史胜率统计
- 策略权重

长期记忆也建议按角色划分。

示例：

- 狼人：悍跳、倒钩、冲票、找神、抗推
- 预言家：上警、跳身份、查验、警徽流
- 女巫：首夜救人、毒人、隐藏身份
- 猎人：身份威慑、开枪目标
- 村民：站边、找狼、票型分析

### 4.4 完整记忆：每局全量归档

完整记忆保存每局游戏的所有内容。

完整记忆不一定直接进入 prompt，但用于回放、复盘、评测、调试和答辩展示。

可保存内容：

- 完整公开日志
- 每个 Agent 每次收到的 observation
- 每次 prompt
- 每次模型原始输出
- 每次解析后的动作
- 每次策略修正
- 每次 fallback
- 每次发言
- 每次投票
- 每次技能
- 最终身份
- 最终胜负
- 赛后复盘结果

## 7. 决策输出结构 ✅

Agent 输出应尽量结构化，便于解析、验证、日志记录和复盘。

示例：

```json
{
  "action_type": "exile_vote",
  "target": 7,
  "choice": null,
  "public_text": "我认为7号发言视角不开，今天优先出7。",
  "private_reasoning": "7号连续跟随2号投票，没有独立找狼逻辑，且攻击预言家时机异常。",
  "confidence": 0.74,
  "alternatives": [2, 10],
  "rejected_reasons": [
    "2号虽然可疑，但目前证据弱于7号",
    "10号发言偏划水，但投票行为暂不构成强狼面"
  ],
  "memory_refs": ["day1_vote_pattern", "day2_speech_7"]
}
```

可记录字段：

- 行动类型
- 目标
- 选择
- 公开发言
- 私有推理
- 置信度
- 候选动作
- 排除理由
- 引用的记忆
- 使用的 skill
- 是否策略修正
- 是否 fallback

## 8. 赛后复盘与评测 ✅

进阶方向可以优先考虑“评测 + 复盘”。

目标不是只看胜负，而是评价每个 Agent 的决策质量。

可评测维度：

- 发言质量
- 投票质量
- 技能使用质量
- 阵营贡献
- 信息一致性
- 是否暴露私有信息
- 是否推进本阵营胜利
- 是否识别关键玩家
- 是否出现明显 bad case

复盘报告可以包括：

- 本局关键转折点
- 每个 Agent 的高光决策
- 每个 Agent 的失误决策
- 女巫是否毒错
- 猎人是否开错枪
- 预言家是否过早或过晚跳身份
- 狼人是否成功冲票
- 村民是否被带偏
- 改进建议

示例：

```text
本局关键失误：
1. 女巫第2晚毒杀5号预言家，导致好人阵营信息链断裂。
2. 9号村民在第3天投票时忽略了7号与2号的连续同票关系。
3. 狼人阵营第1天成功形成冲票，4名狼人中3人投给同一好人。

改进建议：
- 女巫毒人前应优先检查目标是否存在预言家行为特征。
- 村民 Agent 应提高对投票一致性的权重。
- 预言家应在第二天明确公开查验链。
```

## 10. 游戏流程建模 ✅

可以把狼人杀建模成图或状态机。

节点：

- 角色 Agent 节点
- 白天场景节点
- 黑夜场景节点
- 发言节点
- 投票节点
- 技能节点
- 死亡宣判节点
- 胜负判定节点

流程：

```text
白天发言
→ 投票
→ 胜负检测
→ 黑夜行动
→ 死亡宣判
→ 胜负检测
→ 下一天白天
```

结束条件：

- 所有狼人死亡，好人胜利
- 所有好人死亡，狼人胜利
- 好人数量小于等于狼人数量，狼人胜利

## 12. 每个 Agent 的信息组成 ⚠️

> **状态**：部分实现。Observation 系统和信息隔离已完整实现。短期记忆（`AgentMemory`、`FieldNotes`、`PlayerProfile`）、中期经验卡片（`game_analysis.py`）、长期策略整合（`consolidation.py`）均已实现。但 Belief 系统未实现（`agent/core/belief.py` 不存在，`belief_snapshot` 字段为空占位）。

每个 Agent 可以拥有：

- 游戏规则
- 游戏配置
- 自己的身份
- 当前 observation
- 短期记忆：当前 prompt 和本局动态上下文
- 中期记忆：自己的推断、对历史局的总结
- 长期记忆：跨多局沉淀出的策略

注意信息边界：

- 村民不能看到夜间行动细节
- 村民不能看到狼人队友信息
- 狼人不能看到预言家查验结果
- 女巫只能看到自己允许知道的夜间信息
